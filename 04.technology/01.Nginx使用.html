Nginx使用手册 ¶

Nginx是一个高性能的HTTP和反向代理服务器，也是一个IMAP/POP3/SMTP代理服务器。

推荐站点和文章 ¶

  • nginx的配置和使用

  • 配置Nginx）

下载Nginx ¶

先下载Nginx，以windows版为例，下载后niginx压缩包并解压到任意目录，双击nginx.exe，在浏览器中访问http://localhost，如果出现Welcome to nginx!页面则说明成功。

nginx常用命令 ¶

bash
# 打开帮助
nginx -h        
# 检查配置文件是否正确
nginx -t        

# 以下命令均要先启动nginx才能执行

# 停止
nginx -s stop 
# 退出  
nginx -s quit   
# 重新启动（注意不会重新读取配置文件）
nginx -s reopen 
# 重新读取配置文件
nginx -s reload

部署项目到Nginx根目录 ¶

以vue项目为例，使用命令npm run build创建输出文件，将dist文件夹下的所有内容复制到nginx目录下的webapp/内（没有的话自行创建）。

修改nginx目录中的conf/nginx.conf文件，在 http -> server 节点中，修改location节的内容：

bash
location / {
    root   webapp;
    index  index.html index.htm;
}

多个项目部署到Nginx ¶

有时一个Nginx中放了好几个子项目，需要将不同的项目通过不同的路径来访问。

对于项目1而言，修改vue.config.js文件的publicPath：

bash
publicPath: '/vue1/'

对于项目2而言，修改vue.config.js文件的publicPath：

bash
publicPath: '/vue2/'

分别在vue项目根目录中使用命令npm run build创建输出文件，将dist文件夹下的所有内容复制到nginx目录下的webapp/vue1和webapp/vue2内（没有的话自行创建）。

修改nginx目录中的conf/nginx.conf文件，在 http -> server 节点中，修改location节的内容：

bash
location /vue1 {
    root   webapp;
    index  index.html index.htm;
}

location /vue2 {
    root   webapp;
    index  index.html index.htm;
}

在nginx根目录使用命令nginx -s reload即可在浏览器中通过http://localhost/vue1、http://localhost/vue2访问项目1、项目2。

端口代理 ¶

当前后端项目分别部署在同一台机器上时，由于无法使用相同的端口，故后端一般会将端口号设置成不同的值（例如8080），但是当前端向后端请求资源时还要加上端口号，未免显得麻烦，故利用可以nginx将前端的指定路径代理到后端的8080端口上。

在conf/nginx.conf文件中增加location：

bash
location /api {
    proxy_pass http://localhost:8080/api;
}

这样，当前端访问/api路径时，实际上访问的是http://localhost:8080/api路径。

Nginx做代理时X-Forwarded-For信息头的处理 ¶

X-Forwarded-For:简称XFF头，它代表客户端，也就是HTTP的请求端真实的IP，只有在通过了HTTP 代理或者负载均衡服务器时才会添加该项。

参考地址

bash
server
{
  listen 4000;
  location / {
    proxy_pass http://127.0.0.1:3000;
    proxy_redirect off;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header Host $host;
    client_max_body_size 10m;                                    # 允许客户端请求的最大单文件字节数
    client_body_buffer_size 128k;                                # 缓冲区代理缓冲用户端请求的最大字节数，
    proxy_connect_timeout 90;                                    # nginx跟后端服务器连接超时时间(代理连接超时)
    proxy_send_timeout 90;                                       # 后端服务器数据回传时间(代理发送超时)
    proxy_read_timeout 90;                                       # 连接成功后，后端服务器响应时间(代理接收超时)
    proxy_buffer_size 4k;                                        # 设置代理服务器（nginx）保存用户头信息的缓冲区大小
    proxy_buffers 4 32k;                                         # proxy_buffers缓冲区，网页平均在32k以下的设置
    proxy_busy_buffers_size 64k;                                 # 高负荷下缓冲大小（proxy_buffers*2）
    proxy_temp_file_write_size 64k;
  }
}

设置Nginx开机自动启动 ¶

  1. 工具下载URL：https://github.com/winsw/winsw/releases

  2. 下载后将该工具放入Nginx的安装目录下，并且将其重命名为 nginx-service.exe

  3. 在nginx安装目录下新建服务日志文件夹 server-logs 文件夹，用来存放nginx服务相关日志

  4. 在该目录下新建 nginx-service.xml 文件，写入配置信息

xml
<!-- nginx-service.xml -->

<!--文件配置内容如下：重点包括三个，日志文件位置、启动命令、关闭命令；我nginx目录为：D:\nginx-1.18.0，根据自己的目录作相应修改。-->
<service>
    <id>nginx</id>
    <name>nginx</name>
    <description>nginx</description>
    <logpath>D:\nginx-1.18.0\server-logs\</logpath>
    <logmode>roll</logmode>
    <depend></depend>
    <executable>D:\nginx-1.18.0\nginx.exe</executable>
    <stopexecutable>D:\nginx-1.18.0\nginx.exe -s stop</stopexecutable>
</service>
  1. 把nginx加入到windows服务中。内容配置好之后，在nginx安装目录下以管理员运行命令：nginx-service.exe install 就成功将其注册为Windows服务。

注意

如果电脑离线，在执行过程中可能会出现提示安装 .NET Framework 3.5。

Windows10系统离线安装 .NET Framework 3.5的方法

  1. 准备一个Windows10 的 ISO镜像文件，然后鼠标右键选择“装载”。

  2. 装载后在“此电脑”中会显示刚才装载的虚拟光驱“DVD驱动器(K:)”，记录一下这个盘符为“K”，后面会用到。（盘符根据自己的实际情况，不一定是"K"）

  3. 以管理员身份来运行“命令提示符”（点击Windows图标，找到“Windows 系统”，右键选择“更多”，再点击“以管理员身份运行”）或者直接鼠标右键Windows图标，选择“Windows PowerShell（管理员）”来执行如下命令：

shell
dism.exe /online /enable-feature /featurename:netfx3 /Source:K:\sources\sxs

注意

其中的盘符K要改成你自己装载ISO镜像文件后实际的虚拟光驱盘符。

  1. 执行命令后，等待部署进度至100%，提示“操作成功完成”，即表示已经完成了Windows10系统的 .NET Framework 3.5安装。

  2. 打开控制面板，进入“程序”，点击“程序和功能”下面的“启用或关闭Windows 功能”，然后在打开的“Windows功能”窗口中查看“.NET Framework 3.5(包括 .NET 2.0和3.0) ”，如果已经被选中，则证明已经安装成功。

常见问题及解答（FAQ） ¶

1、Vue项目打包使用Nginx发布后刷新页面报404错误 ¶

问题现象：

使用Vue-cli构建的项目，在本地调试运行都是OK的，打包之后使用Nginx发布后，指定跳转页面是ok的，但是刷新页面，reload页面时，报错404，找不到页面。Nginx端日志显示找不到资源路径。

解决方案：

方案1 ¶

首先，出问题的原因是：在 Vue 项目中，可以选择 hash或者 history.pushState() 实现路由跳转。

使用history模式的配置是：

js
export default new Router({
    mode: 'history',
})

那么，当我们 npm run build 完成并部署到服务器后，刷新某个路由下的页面，会出现 404 或者 502 错误。

这是因为刷新页面时访问的资源在服务端找不到，因为vue-router设置的路径不是真实存在的路径。所以把mode改为hash模式就可以了。

方案2 ¶

继续使用history模式，修改Nginx的配置，使所有的请求都定向到index.html页面

bash
  location / {
		try_files $uri $uri/ @router;
		index index.html;
	}
	location @router {
		rewrite ^.*$ /index.html last;
  }

2、NGINX 报错 413 Request Entity Too Large 解决方案 ¶

问题原因

用Nginx反向代理服务器，进行附件相关的操作时，当文件大小超过1M，会出现413 Request Entity Too Large，这是由于nginx客户端默认的最大请求体是1M。

解决方案

通过修改Nginx配置文件（nginx.conf）的“client_max_body_size”属性来解决。

在http节点下（http全局） ¶

bash
http {
    # 将nginx代理的所有请求实体的大小限制为1024m
    client_max_body_size 1024m;
}

在server节点下（server全局） ¶

bash
server {
    # 将该服务下的所有请求实体的大小限制为1024m
    client_max_body_size 1024m;
}

在location节点下（单应用） ¶

bash
location /test {
    # 将此路由请求的实体大小限制为1024m
    client_max_body_size 1024m;
}
